# Context Factory

Purpose: compress the project into agent-usable context.

## Include

- product goal
- current architecture
- coding patterns
- brand/taste rules
- security rules
- database rules
- "do not touch" areas
- known failure modes
- definition of good work

## Core Idea

Agents are only as good as the context bubble they operate inside.

## Output

A context bubble should be short enough to use during implementation and specific enough to prevent unsafe guessing.

It should describe what matters for this project, not generic engineering advice.
